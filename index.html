Metasploit MCP Server Documentation

Welcome to the documentation for the Metasploit Model Context Protocol (MCP) Server.

This server allows you to interact with Metasploit Framework programmatically using the MCP standard.

Overview

The server exposes several tools and endpoints to control Metasploit, manage sessions, generate payloads, and run modules.

Connecting to the Server

The MCP server communicates over Server-Sent Events (SSE) and HTTP POST messages.

  • SSE Endpoint: /mcp/sse (for receiving messages from the server)
  • Messages Endpoint: /mcp/messages/ (for sending messages to the server)

Clients should first establish an SSE connection to /mcp/sse. Then, they can send MCP-formatted JSON messages via HTTP POST to /mcp/messages/.

Available Tools (via MCP)

The server provides a range of tools that can be invoked through MCP messages. Refer to the server's capabilities (usually announced on connection) or the OpenAPI documentation (/docs) for a detailed list of tools, their parameters, and expected responses. Examples include:

  • list_exploits
  • list_payloads
  • generate_payload
  • run_exploit
  • list_active_sessions
  • send_session_command
  • start_listener
  • stop_job
  • And many more...

Health Check

A health check endpoint is available to verify the server's status and its connection to the Metasploit RPC service:

  • GET /healthz

API Documentation (OpenAPI)

For detailed information about each tool, its parameters, and response schemas, please refer to the auto-generated OpenAPI (Swagger) documentation available at:

  • GET /docs

Configuration

The server is configured using environment variables. Key variables include:

  • MSF_SERVER: Metasploit RPC server address (default: 127.0.0.1)
  • MSF_PORT: Metasploit RPC server port (default: 55553)
  • MSF_PASSWORD: Metasploit RPC password
  • MSF_SSL: Use SSL for MSF RPC connection (default: false)
  • PAYLOAD_SAVE_DIR: Directory to save generated payloads
  • LOG_LEVEL: Logging level (default: INFO)
  • LISTEN_PORT: Port for this MCP server to listen on (default: 8085)

When running in Docker, these can be passed during docker run.
